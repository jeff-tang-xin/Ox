fix(prompt): plug over-exploration & missing-action leaks in system prompt

Two recurring failure modes were traced back to the prompt-injection
layer itself, not the LLM:

1. **Over-exploration** — UNIFIED_CORE_CODING preached "build a
   relationship model" but gave no hard budget or stop rule; the agent
   kept file_read'ing the same file 5+ times, or restarted exploration
   after an impact-gate warning.

2. **Missing action** — the tool-block in every prompt variant listed
   a *subset* of legal actions (build_unified_tool_blocked
   code_graph / delete_range / project_detect / read_symbol;
   build_tool_block missed code_graph / project_detect). The LLM had
   no authoritative whitelist and improvised (start_line vs
   start_anchor, etc.).

Fix:

  crates/ox-core/src/agent/unified_action.rs
    + pub const UNIFIED_ACTIONS_LIST — single source of truth listing
      all 18 legal actions. Prompt blocks now embed this constant.

  crates/ox-core/src/context/system_prompt.rs
    * build_unified_tool_block — embeds UNIFIED_ACTIONS_LIST header,
      splits tool grammar across 3 lines covering all 18 actions,
      adds anti-example for delete_range start_line/end_line.
    * build_tool_block — same treatment for the non-unified variant.
    * METHODOLOGY — adds 3 hard rules:
        - single-turn explore budget ≤ 12
        - implementation phase (finding confirmed / files_modified)
          forbids reopening new exploration chains
        - impact-gate warning → call code_graph impact once, resume
          the ORIGINAL edit, do not restart exploration
    * UNIFIED_CORE_CODING — header-line "explore starts with
      code_graph list_repos/query" plus explicit whitelist-only rule
      and start_anchor/end_anchor callout.

cargo check --workspace: 0 error.
