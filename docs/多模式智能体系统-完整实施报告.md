## 📊 约束规则实施详情

### Spec Mode 工作流（6步）

| 步骤 | 名称 | 工具权限 | 代码修改 | 用户确认 | 说明 |
|------|------|---------|---------|---------|------|
| 1 | Requirement Analysis | ✅ | ❌ | ❌ | 分析任务，阅读代码 |
| 2 | Generate spec.md | ✅ | ❌ | ❌ | 创建规范文档 |
| 3 | Await Spec Confirmation | ❌ | ❌ | ✅ | 等待用户确认 |
| 4 | Generate task.md | ✅ | ❌ | ❌ | 创建任务计划 |
| 5 | Await Task Confirmation | ❌ | ❌ | ✅ | 最终确认 |
| 6 | Execute Code | ✅ | ✅ | ❌ | **执行代码修改** |

**关键约束**:
- ✅ Step 1-5: 可以使用 `file_read`, `file_search`, `code_search` 等工具
- ❌ Step 1-5: **禁止**修改源代码文件（.rs, .py, .js, .ts 等）
- ✅ Step 1-5: **允许**创建/修改文档文件（.md, .txt, .json 配置等）
- ✅ Step 6: 完全开放，可以修改任何文件

---

### Council Mode 工作流（6步）

| 步骤 | 名称 | 工具权限 | 代码修改 | 用户确认 |
|------|------|---------|---------|---------|
| 1 | Topic Definition | ✅ | ❌ | ✅ |
| 2 | Proposal Phase | ✅ | ❌ | ❌ |
| 3 | Review Phase | ✅ | ❌ | ❌ |
| 4 | Rebuttal Phase | ✅ | ❌ | ❌ |
| 5 | Arbitration | ✅ | ❌ | ❌ |
| 6 | Conclusion | ✅ | ❌ | ✅ |

**特点**:
- ✅ 所有步骤都可以使用工具进行研究
- ❌ 所有步骤都**禁止**代码修改
- Council 模式纯用于决策，不包含代码执行

### 1. 代码修改检测

在 `WorkflowEngine` 中提供检查方法：

```rust
impl WorkflowEngine {
    /// Check if code file modification is allowed in current step
    pub fn allows_code_modification(&self) -> bool {
        if let Some(step) = self.current_step() {
            step.allow_code_modification
        } else {
            false
        }
    }
}
```
