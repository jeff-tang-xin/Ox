# UI 优化改进说明

## 概述
本次优化针对 Ox CLI 的终端用户界面进行了全面改进，提升了用户体验和视觉清晰度。

## 优化内容

### 1. 布局比例优化 ✅
**问题**: 输入区域固定3行高度，对于长输入不够友好；header占用过多空间。

**改进**:
- Header 最大高度从4行减少到3行，节省垂直空间
- 输入区域动态调整：普通模式2行，确认模式3行
- 主输出区域最小高度设置为5行，确保有足够显示空间
- 整体布局更加合理，给用户更多对话显示空间

**代码位置**: `crates/ox-cli/src/terminal/render.rs` - `render()` 函数

### 2. 消息类型视觉区分增强 ✅
**问题**: 不同消息类型之间区分度不够明显，用户难以快速识别。

**改进**:
- **工具调用 (Tool)**: 添加淡黄色背景 `Rgb(40, 40, 30)`，图标更醒目
- **工具结果 (ToolResult)**: 
  - 成功：绿色背景 `Rgb(20, 40, 20)` + ✓ 图标
  - 失败：红色背景 `Rgb(50, 20, 20)` + ✗ 图标
- **错误消息 (Error)**: 深红色背景 `Rgb(60, 10, 10)` + 粗体显示
- **系统消息 (System)**: 保持简洁的绿色指示器
- **用户消息 (User)**: 保持高亮青色图标

**代码位置**: `crates/ox-cli/src/terminal/render.rs` - `render_single_line()` 函数

### 3. 滚动指示器改进 ✅
**问题**: 用户滚动后没有清晰的视觉反馈。

**改进**:
- 滚动时标题显示具体行数：`Ox ↓ 5 lines up`
- 滚动标题使用黄色高亮，更容易注意到
- 底部状态栏简化，移除冗余信息

**代码位置**: `crates/ox-cli/src/terminal/render.rs` - `render_chat()` 函数

### 4. 状态栏简化 ✅
**问题**: 底部状态栏信息过多，显得杂乱。

**改进**:
- 左侧：只显示模型名称和工作目录（最关键信息）
- 右侧：紧凑格式显示消息数量和成本
- 移除了 TRUSTED 提示（可通过 `/trust` 命令查看）
- 使用 `|` 替代 `│` 分隔符，更简洁

**之前**: `model | dir | TRUSTED /untrust | ⏳ msgs │ cost`
**现在**: `model | dir | ⏳ msgs | cost`

**代码位置**: `crates/ox-cli/src/terminal/render.rs` - `render_status_bar()` 函数

### 5. 输入提示优化 ✅
**问题**: Y/N/T 确认模式和普通输入模式切换不够直观。

**改进**:
- **确认模式**: 
  - 黄色边框和标题 "Confirmation Mode"
  - 提示符变为 `❯ Y/N/T > `（黄色粗体）
  - 视觉上明确区分于普通输入
- **Agent 运行中**: 
  - 提示符变为 `▸ ox > `（流式颜色）
  - 与 spinner 动画颜色一致
- **普通模式**: 
  - 保持蓝色 `ox > ` 提示符

**代码位置**: `crates/ox-cli/src/terminal/render.rs` - `render_input_pane()` 函数

## 技术细节

### 颜色方案
- **背景色**: 保持黑色 `Rgb(0, 0, 0)`
- **输入背景**: 深灰 `Rgb(30, 30, 30)`
- **工具背景**: 淡黄 `Rgb(40, 40, 30)`
- **成功背景**: 深绿 `Rgb(20, 40, 20)`
- **错误背景**: 深红 `Rgb(50, 20, 20)` / `Rgb(60, 10, 10)`

### 布局约束
```rust
Layout::vertical([
    Constraint::Length(header_height),      // 2-3 行
    Constraint::Min(5),                      // 至少 5 行
    Constraint::Length(1),                   // 状态栏 1 行
    Constraint::Length(input_height),        // 2-3 行
])
```

## 用户体验提升

1. **更快的信息识别**: 通过颜色编码，用户可以快速区分消息类型
2. **更清晰的导航**: 滚动指示让用户知道当前位置
3. **更少的视觉干扰**: 简化的状态栏减少认知负担
4. **更直观的交互**: 确认模式的视觉提示避免误操作
5. **更多的对话空间**: 优化的布局提供更多消息显示区域
