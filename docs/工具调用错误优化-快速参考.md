# 工具调用错误优化 - 快速参考

## 🎯 问题与解决方案

### 问题 1: LLM 生成的 JSON 参数格式错误

**常见表现**:
```
Invalid tool arguments JSON: expected `,` or `}` at line 1 column 50
```

**根本原因**:
- Token limit 导致 JSON 被截断
- LLM 生成了不规范的 JSON（尾随逗号、未转义字符等）

**解决方案**: ✅ 已实施
- 提供详细的 JSON 语法修复建议
- 针对每个工具提供正确的示例
- 使用 emoji 图标提高可读性

**示例对比**:

| 优化前 | 优化后 |
|--------|--------|
| `Invalid tool arguments JSON: expected ',' or '}'` | `❌ JSON Parse Error for tool 'file_write':`<br>`💡 How to fix:`<br>`• Ensure valid JSON syntax`<br>`• Quote all keys with double quotes`<br>`📝 Example: {"path": "test.txt"}` |

## 💡 LLM 使用建议

### 生成 JSON 参数时的注意事项

1. **JSON 格式要求**:
   ```json
   ✅ 正确: {"path": "src/main.rs", "limit": 100}
   ❌ 错误: {path: "src/main.rs", "limit": 100,}  // 键未加引号，尾随逗号
   ```

2. **特殊字符转义**:
   ```json
   ✅ 正确: {"content": "Hello\nWorld"}
   ❌ 错误: {"content": "Hello
   World"}
   ```

3. **括号配对**:
   ```json
   ✅ 正确: {"edits": [{"old": "a", "new": "b"}]}
   ❌ 错误: {"edits": [{"old": "a", "new": "b"}  // 缺少 ]
   ```

### 工具选择指南

- **读取文件**: 使用 `file_read`
- **写入新文件**: 使用 `file_write`
- **修改现有文件**: 使用 `file_patch`（更高效）
- **查找文件**: 使用 `file_search`
- **搜索代码内容**: 使用 `code_search`
- **执行系统命令**: 使用 `shell_exec`（需要确认）

## 🔍 调试方法

### 查看详细日志

```bash
# 实时查看 Ox 日志
tail -f ~/.ox/logs/ox.log

# 只看工具调用错误
```

### 常见错误模式识别

| 日志关键词 | 问题类型 | 解决方法 |
|-----------|---------|---------|
| `expected ',' or '}'` | JSON 格式错误 | 检查括号和逗号 |
| `missing field` | 缺少参数 | 添加必需参数 |
| `Unknown tool` | 工具名称错误 | 检查工具列表 |
| `No such file` | 文件路径错误 | 验证文件路径 |
| `Permission denied` | 权限问题 | 检查文件权限 |
