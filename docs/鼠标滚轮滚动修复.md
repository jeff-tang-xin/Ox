# 鼠标滚轮滚动修复指南

## 🐛 问题描述

**现象**: 在 Windows 上,鼠标滚轮无法在 chat 内容区域滚动  
**影响平台**: Windows (cmd/powershell)  
**根本原因**: 鼠标捕获被条件编译禁用

---

## 🔍 问题分析

### 代码位置
`F:\rust\Ox\crates\ox-cli\src\main.rs` 第 350-353 行

### 原始代码
```rust
// Enable mouse capture for scroll events (skip on Windows where it
// can cause garbled input in cmd/powershell).
#[cfg(not(target_os = "windows"))]
stdout.execute(EnableMouseCapture)?;
```

**问题**: 
1. `#[cfg(not(target_os = "windows"))]` 导致 Windows 上完全跳过鼠标启用
2. 导入也有条件编译 (第 11-12 行)
3. 注释提到"会导致输入混乱",但这是旧版本 crossterm 的问题

### 为什么之前禁用?

**历史原因** (可能):
- 旧版 crossterm 在 Windows Terminal 上有兼容性问题
- cmd.exe 和 PowerShell 的鼠标事件处理不完善
- 可能导致键盘输入和鼠标事件冲突

**现状**:
- ✅ crossterm 0.27+ 已修复 Windows 鼠标支持
- ✅ Windows Terminal 1.0+ 完美支持鼠标事件
- ✅ 现代 PowerShell 7+ 无兼容性问题

## 📊 工作原理

### 鼠标事件流程

```
用户滚动鼠标滚轮
  ↓
crossterm 捕获 MouseEvent
  ↓
EventHandler 转换为 Event::ScrollUp/ScrollDown
  ↓
主循环接收事件 (main.rs:931-952)
  ↓
检查鼠标是否在 chat 区域内
  ↓
调用 app.scroll_up(3) 或 app.scroll_down(3)
  ↓
更新 scroll_offset
  ↓
标记 dirty = true
  ↓
下一帧渲染时应用新的滚动位置
```

### Chat 区域检测

```rust
// main.rs:931-939
Some(Event::ScrollUp { column, row }) => {
    // Only handle scroll if mouse is in chat area
    if let Some((cx, cy, cw, ch)) = app.chat_area {
        if column >= cx && column < cx + cw && row >= cy && row < cy + ch {
            app.scroll_up(3);
            app.user_scrolled = true;
            app.dirty = true;
        }
    }
}
```

**chat_area 设置** (render.rs:106):
```rust
app.chat_area = Some((area.x, area.y, area.width, area.height));
```

## ⚠️ 已知限制

### 1. Windows Terminal 要求

**最低版本**: Windows Terminal 1.0+  
**推荐**: 最新版 Windows Terminal

**检查版本**:
```powershell
wt --version
```

**如果使用的是旧版 cmd.exe**:
- 可能仍有兼容性问题
- 建议切换到 Windows Terminal

### 2. 远程桌面/SSH

通过 RDP 或 SSH 连接时:
- 鼠标事件可能无法正确传递
- 建议使用键盘快捷键 (PageUp/PageDown)

### 3. 虚拟机

在某些 VM 中:
- VMware/VirtualBox 可能需要启用"鼠标集成"
- Parallels (Mac) 通常工作正常

## 🔧 故障排除

### 问题 1: 编译错误 "cannot find value `EnableMouseCapture`"

**原因**: 导入语句未更新  
**解决**: 确保第 7 行包含 `EnableMouseCapture, DisableMouseCapture`

### 问题 2: 滚动不工作

**检查清单**:
1. ✅ 鼠标在 chat 区域内 (不是输入框或侧边栏)
2. ✅ 有足够的内容可以滚动


### 问题 3: 滚动太快/太慢

**调整滚动速度** (main.rs:935, 945):

```rust
// 当前: 每次滚动 3 行
app.scroll_up(3);
app.scroll_down(3);

// 改为更快的滚动
app.scroll_up(5);  // 每次 5 行
app.scroll_down(5);
```

### 问题 4: 键盘输入混乱

**症状**: 按键盘时出现奇怪字符  
**原因**: 鼠标捕获与键盘冲突 (极少见)  

**临时解决**:
```rust
// 在 main.rs 中暂时禁用
// stdout.execute(EnableMouseCapture)?;  // 注释掉
```

**永久解决**:
- 更新 Windows Terminal 到最新版
- 重启终端会话

```rust
pub fn scroll_up(&mut self, delta: u16) {
    self.scroll_offset = self.scroll_offset.saturating_add(delta);
}

pub fn scroll_down(&mut self, delta: u16) {
    self.scroll_offset = self.scroll_offset.saturating_sub(delta);
}

pub fn scroll_to_bottom(&mut self) {
    self.scroll_offset = 0;
```
