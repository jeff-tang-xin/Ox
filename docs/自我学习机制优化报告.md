### 1.2 增强记忆提取日志

**修改文件**: `crates/ox-core/src/memory/mod.rs`

**变更**：
```rust
pub fn update_from_turn(...) {
    // Track successful tool calls for learning
    let mut successful_tools = Vec::new();
    
    for msg in messages {
        // Extract knowledge from tool calls
        for tc in tool_calls {
            if let Some(node) = MemoryNode::extract_from_tool_call(...) {
                self.store(node);
                successful_tools.push(tc.name.clone());
            }
        }
        
        // Extract knowledge from conversation content
        if let Some(node) = MemoryNode::extract_from_conversation(...) {
            self.store(node);
        }
    }
    
    // Log learning statistics
    if !successful_tools.is_empty() {
        tracing::debug!(
            "[LEARNING] Extracted {} memories from tools: {:?}",
            successful_tools.len(),
            successful_tools
        );
    }
}
```

**收益**：
- 可以看到每次对话学到了什么
- 便于调试和优化记忆提取策略

## 📊 当前自我学习机制

### 核心流程

```
用户交互 → 工具调用 → 自动提取知识 → 存储到记忆
     ↓
显式反馈 (/feedback good) → 强化相关记忆 → 提高深度和权重
     ↓
隐式信号 (代码接受率) → EMA 追踪 → 调整满意度评分
     ↓
记忆检索 → 注入上下文 → LLM 使用知识 → 更好的回答
```

### 记忆类型

| 类型 | 说明 | 初始深度 | 示例 |
|------|------|---------|------|
| **Fact** | 项目事实 | 1 | "使用 PostgreSQL 数据库" |
| **Style** | 编码风格 | 3 | "偏好函数式编程" |
| **Architectural** | 架构决策 | 2 | "采用分层架构" |
| **AntiPattern** | 反模式 | 2 | "避免在循环中查询数据库" |
| **Business** | 业务逻辑 | 2 | "用户必须有邮箱验证" |
| **BestPractice** | 最佳实践 | 2 | "使用 Result 处理错误" |
| **Pattern** | 设计模式 | 2 | "使用 Repository 模式" |
| **MetaSkill** | 元技能 | 3 | "先写测试再实现功能" |

### 知识提取来源

1. **工具调用** (`extract_from_tool_call`)
   - `file_write` → 提取文件路径、内容结构
   - `shell_exec` → 提取命令模式、输出格式
   - `code_search` → 提取搜索模式、代码结构

2. **对话内容** (`extract_from_conversation`)
   - 用户指令 → 提取 Style 记忆
   - 架构讨论 → 提取 Architectural 记忆
   - Bug 修复 → 提取 AntiPattern 记忆

3. **显式反馈** (`/feedback good`)
   - 检索最近 5 条相关记忆
   - 增加深度（depth++）
   - 更新最后访问时间
