# 优化建议

## 1. Session.file_handle 生命周期管理

**问题**: `Session` 结构体新增了 `file_handle: Option<BufWriter<File>>`，用于复用文件句柄避免重复打开。但当前没有 `Drop` 实现，文件可能不会正确 flush 和关闭。

**建议**: 为 `Session` 实现 `Drop` trait，确保资源正确释放：

```rust
impl Drop for Session {
    fn drop(&mut self) {
        if let Some(ref mut writer) = self.file_handle {
            let _ = write);
        }
    }
}
```

**状态**: ✅ 已修复 — `Session` 已实现 `Drop` trait，确保 `file_handle` 正确 flush。

**影响**: 防止崩溃或非正常退出时数据丢失。

## 2. 渲染缓存内存上限

**问题**: `OutputPane` 的 `rendered_cache: Vec<Option<Vec<Line<'static>>>>` 会随着消息数量增长而增长，虽然 `lines` 有 `MAX_LINES = 2000` 限制，但缓存的 `Line<'static>` 持有 owned 字符串，内存占用可能较大。

**建议**: 
- 方案 A: 设置缓存条目数上限（如只缓存最近 500 条已渲染行）
- 方案 B: 当 `lines` 被 `trim_excess()` 裁剪时，同步清理对应的缓存（当前已做，确认逻辑正确即可）

**优先级**: 中 — 正常使用场景下不易触发，但长会话可能积累较多内存。

---

## 3. user_scrolled 自动恢复逻辑

**问题**: 当前 `user_scrolled` 只在 `scroll_offset == 0`（即用户主动滚到底部）时才会重置为 `false`。如果用户滚动到中间后不再操作，后续新消息会在底部显示但用户看不到。

**建议**: 在收到 `AgentToUiEvent` 新内容时，增加智能恢复逻辑：

```rust
// 当用户未手动滚动时，始终自动滚动
// 当用户滚动超过一定阈值时，显示提示但不再自动滚动
// 当用户接近底部（如最后 3 行）时，自动恢复
if !app.user_scrolled || app.scroll_offset < 3 {
    app.scroll_to_bottom();
    app.user_scrolled = false;
}
```

**状态**: ✅ 已修复 — `scroll_offset < 3` 时自动恢复 `user_scrolled = false`。

**影响**: 改善用户长时间不交互后的体验。

## 4. Cargo.lock CRLF 警告

**问题**: Git 输出显示以下警告：
```
warning: in the working copy of 'Cargo.lock', LF will be replaced by CRLF the next time Git touches it
```

**建议**: 在项目根目录添加或更新 `.gitattributes` 文件：

```gitattributes
# Lock files should always use LF
*.lock text eol=lf

# Rust source files
*.rs text eol=lf

# Config files
*.toml text eol=lf
*.json text eol=lf
*.md text eol=lf
```

**影响**: 统一换行符，避免跨平台协作时的 Git 混乱。

**状态**: ✅ 已修复 — 已创建 `.gitattributes` 文件，配置 `*.lock`, `*.rs`, `*.toml` 等使用 `eol=lf`。

## 5. 日志文件句柄泄漏风险（低优先级）

**问题**: `main.rs` 中日志初始化使用 `std::fs::OpenOptions::new().open()` 打开文件，通过 `Mutex` 包装传给 `tracing_subscriber`。文件句柄在程序退出时由 OS 回收，但没有显式关闭。

**建议**: 可将 `log_file` 包装在 `Arc<Mutex<...>>` 中并在合适时机手动 drop，或接受程序退出时 OS 自动清理的行为（当前做法可接受）。

**优先级**: 低 — 程序生命周期内文件句柄不会重复打开。
