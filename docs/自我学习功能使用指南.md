# 自我学习功能使用指南

## 🎯 核心理念

**让 Ox 通过交互持续学习，越来越懂你的项目**

---

## 📚 主要命令

### 1. `/feedback good` - 强化有用知识

**何时使用**：
- Ox 的回答很有帮助
- 代码建议符合你的风格
- 解决方案解决了问题

**效果**：
```bash
User: "如何优化这个函数？"
Assistant: [提供优化建议]
User: "/feedback good"

✅ Feedback noted: positive. Related memories reinforced.
```

**后台发生**：
- 检索最近对话相关的 5 条记忆
- 增加这些记忆的深度（depth++）
- 更新最后访问时间
- 提高在后续检索中的权重

---

### 2. `/memory` - 查看学习成果

**何时使用**：
- 想了解 Ox 学到了什么
- 检查知识库是否准确
- 调试记忆系统

**输出示例**：
```
📚 Learning Statistics:
Project memories: 42 | Global memories: 128

Memories by type:
  📝 fact: 15
  🏗️ architectural: 10
  ✅ best_practice: 8
  🎨 style: 5
  ⚠️ anti_pattern: 3
  🧠 meta_skill: 1

Recent memories:
  [project] Use tokio for async runtime (depth: 3)
  [global] Prefer functional programming patterns (depth: 2)
  ...
```

**解读**：
- **Project memories**: 当前项目特定的知识
- **Global memories**: 跨项目的通用知识
- **depth**: 记忆的重要性（1-5），越高越重要
- **类型分布**: 了解 Ox 主要学到了什么类型的知识

---

### 3. `/remember <内容>` - 手动添加知识

**何时使用**：
- 有重要的项目信息需要记住
- Ox 没有自动提取到关键知识
- 想明确告诉 Ox 某个规则

**示例**：
```bash
/remember 项目使用 PostgreSQL 14，端口 5432
/remember 所有 API 必须返回 JSON 格式
/remember 禁止使用 unwrap()，统一用 ? 运算符
```

**效果**：
- 创建一条 Style 类型的记忆
- 初始深度为 3（较高优先级）
- 立即存入数据库

---

### 4. `/forget <关键词>` - 删除错误知识

**何时使用**：
- Ox 学到了错误的信息
- 过时的知识需要清理
- 想移除某类记忆

**示例**：
```bash
/forget MySQL          # 删除所有包含 "MySQL" 的记忆
/forget old_api        # 删除包含 "old_api" 的记忆
```

**效果**：
- 搜索匹配关键词的记忆
- 从数据库中删除
- 显示删除数量

---

## 💡 最佳实践

### ✅ DO（推荐做法）

1. **及时反馈**
   ```bash
   # Ox 提供了好的建议后
   /feedback good
   
   # 而不是等到对话结束
   ```

2. **定期查看学习成果**
   ```bash
   # 每周运行一次
   /memory
   
   # 检查是否有错误的记忆
   ```

3. **主动教授关键知识**
   ```bash
   # 项目启动时
   /remember 这是一个 Rust 项目，使用 Cargo 管理依赖
   /remember 代码风格遵循 rustfmt 默认配置
   ```

4. **纠正错误**
   ```bash
   # 如果 Ox 记住了错误的信息
   /forget 错误的关键词
   /remember 正确的信息
   ```

---

### ❌ DON'T（避免做法）

1. **不要过度使用 `/feedback good`**
   - 只在真正有帮助时使用
   - 频繁反馈会降低信号质量

2. **不要删除太多记忆**
   - `/forget` 要谨慎使用
   - 可能删除有用的知识

3. **不要期望即时效果**
   - 学习是渐进的过程
   - 需要多次交互才能建立可靠的知识库

## 🔍 常见问题

### Q1: Ox 学到了什么？

**A**: 运行 `/memory` 查看。你会看到：
- 记忆总数
- 按类型分类的统计
- 最近的记忆列表

### Q2: 如何让 Ox 记住某个规则？

**A**: 两种方式：
1. **显式**：`/remember 规则内容`
2. **隐式**：在对话中提到，Ox 会自动提取

### Q3: Ox 会忘记知识吗？

**A**: 会的，通过两种机制：
1. **衰减**：长时间不访问的记忆会被降低权重
2. **Janitor**：定期清理低价值的记忆（depth ≤ 1 且超过 30 天未访问）

### Q4: 如何让 Ox 更懂我的项目？

**A**: 
1. **多交互**：每次对话都会自动学习
2. **给反馈**：`/feedback good` 强化有用的知识
3. **教关键信息**：`/remember` 添加重要规则
4. **耐心等待**：学习是累积的过程

### Q5: 记忆会影响回答吗？

**A**: 会的！记忆会在每次对话时：
1. 检索相关记忆（最多 5-8 条）
2. 注入到 System Prompt
3. LLM 基于这些知识生成回答

## 📊 学习流程示例

### 场景：学习项目的数据库配置

**第 1 次交互**：
```
User: "项目用什么数据库？"
Assistant: [读取 config.toml] "PostgreSQL 14"
User: "/feedback good"

→ 学到：MemoryNode { content: "使用 PostgreSQL 14", type: Fact, depth: 2 }
```

**第 2 次交互**（一周后）：
```
User: "如何连接数据库？"
Assistant: [检索到 PostgreSQL 记忆]
           "使用 postgres crate，连接字符串：postgresql://localhost:5432/mydb"
User: "/feedback good"

→ 强化：depth: 2 → 3
```

**第 3 次交互**（一个月后）：
```
User: "帮我写一个查询函数"
Assistant: [检索到 PostgreSQL 记忆，depth=3，高优先级]
           "使用 sqlx::query! 宏，类型安全的查询..."
           
→ 无需反馈，已经非常准确
```

## 🎉 总结

**自我学习的核心**：
1. **自动提取** - 每次对话都学习
2. **显式强化** - `/feedback good` 标记有用知识
3. **智能检索** - 根据上下文注入相关知识
4. **持续优化** - 时间越长，越懂你的项目

**开始使用**：
```bash
# 1. 查看当前学习成果
/memory

# 2. 下次 Ox 帮到你时
/feedback good

# 3. 定期检查和优化
/forget <过时关键词>
/remember <重要规则>
```

享受 Ox 越来越懂你的过程！🚀
