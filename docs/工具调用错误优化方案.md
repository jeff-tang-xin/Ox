# 工具调用错误优化方案

## 🐛 当前问题分析

### 1. JSON 参数解析失败（最常见）

**现象**:
```
Invalid tool arguments JSON: expected `,` or `}` at line 1 column 50
```

**原因**:
- LLM 生成的 JSON 被截断（token limit 导致）
- LLM 生成不规范的 JSON（如 trailing commas, unquoted keys）
- 特殊字符未正确转义

**当前代码位置**: `crates/ox-core/src/agent/mod.rs:486-512`

---

### 2. 缺少必填参数

**现象**:
```
Missing required parameter: path
```

**原因**:
- LLM 忘记提供必需参数
- 参数名称拼写错误
- 参数值为 null 或空字符串

---

### 3. 文件操作错误

**现象**:
```
Failed to read /path/to/file: No such file or directory (os error 2)
Permission denied (os error 13)
Path validation failed: Path is outside working directory
```

**原因**:
- 文件不存在或路径错误
- 权限不足
- 路径穿越保护触发

---

### 4. Shell 命令执行失败

**现象**:
```
Command timed out after 30000ms. Process killed.
[exit code: 1]
[stderr] bash: command not found
```

**原因**:
- 命令执行超时（默认 30s）
- 命令返回非零退出码
- 命令不存在或环境变量问题

---

### 5. 工具不存在

**现象**:
```
Unknown tool: 'file_edit'. Available tools: file_read, file_write, ...
```

**原因**:
- LLM 使用了错误的工具名称
- 工具名称拼写错误

## ✅ 优化方案

### 方案 1: 增强 JSON 解析容错性 ⭐⭐⭐

#### 问题
LLM 经常生成不规范的 JSON，当前实现直接拒绝。

#### 解决方案
添加 JSON 修复逻辑，尝试自动修正常见问题：

```rust
// 在 crates/ox-core/src/agent/mod.rs 中修改
fn parse_tool_arguments(raw_args: &str, tool_name: &str) -> Result<serde_json::Value, String> {
    let trimmed = raw_args.trim();
    
    if trimmed.is_empty() {
        return Ok(serde_json::Value::Object(serde_json::Map::new()));
    }
    
    // 第一次尝试：标准解析
    if let Ok(value) = serde_json::from_str::<serde_json::Value>(trimmed) {
        return Ok(value);
    }
    
    // 第二次尝试：修复常见 JSON 问题
    tracing::warn!(
        "Standard JSON parse failed for '{}', attempting repair",
        tool_name
    );
    
    let repaired = repair_json(trimmed);
    
    match serde_json::from_str::<serde_json::Value>(&repaired) {
        Ok(value) => {
            tracing::info!("JSON repair succeeded for '{}'", tool_name);
            Ok(value)
        }
        Err(e) => {
            Err(format!(
                "Invalid tool arguments JSON: {e}\n\n\
                 Original: {}\n\n\
                 Please provide valid JSON with correct syntax.",
                if trimmed.len() > 200 {
                    format!("{}...(truncated)", &trimmed[..200])
                } else {
                    trimmed.to_string()
                }
            ))
        }
    }
}

/// Attempt to repair common JSON issues
fn repair_json(input: &str) -> String {
    let mut result = input.to_string();
    
    // 1. Remove trailing commas before } or ]
    result = regex::Regex::new(r",\s*([}\]])")
        .ok()
        .map(|re| re.replace_all(&result, "$1").to_string())
        .unwrap_or(result);
    
    // 2. Fix unquoted keys (simple case: word followed by :)
    result = regex::Regex::new(r"(\w+)\s*:")
        .ok()
        .map(|re| re.replace_all(&result, "\"$1\":").to_string())
        .unwrap_or(result);
    
    // 3. Replace single quotes with double quotes (outside strings)
    // This is tricky, so we skip it for now
    
    result
}
```

**依赖添加** (`crates/ox-core/Cargo.toml`):
```toml
regex = { workspace = true }
```

### 方案 2: 提供更友好的错误提示 ⭐⭐⭐

#### 问题
当前错误信息对 LLM 不够友好，难以指导其修正。

#### 解决方案
为每种错误类型提供详细的修复建议：

```rust
// 在 agent/mod.rs 的工具执行部分
let args: serde_json::Value = match parse_tool_arguments(&tc.arguments, &tc.name) {
    Ok(v) => v,
    Err(error_msg) => {
        // Provide helpful guidance
        let guidance = match tc.name.as_str() {
            "file_read" => "\n\nExample: {\"path\": \"src/main.rs\", \"limit\": 100}",
            "file_write" => "\n\nExample: {\"path\": \"output.txt\", \"content\": \"Hello World\"}",
            "shell_exec" => "\n\nExample: {\"command\": \"ls -la\", \"timeout_ms\": 5000}",
            _ => "",
        };
        
        let error_msg = format!(
            "❌ Tool argument error for '{}':\n{}\n\n\
             💡 How to fix:\n\
             - Ensure all required parameters are provided\n\
             - Use valid JSON syntax (no trailing commas)\n\
             - Quote all string values with double quotes\n\
             {}\n\n\
             Please retry with corrected arguments.",
            tc.name, error_msg, guidance
        );
        
        let result_msg = Message::ToolResult {
            tool_call_id: tc.id.clone(),
            content: error_msg.clone(),
        };
        new_messages.push(result_msg.clone());
        messages.push(result_msg);
        
        let _ = ui_tx.send(AgentToUiEvent::ToolResult {
            name: tc.name.clone(),
            output: error_msg,
            is_error: true,
        });
        continue;
    }
};
```

### 方案 3: 添加参数预验证 ⭐⭐

#### 问题
LLM 提供的参数可能在执行前就可以检测到问题。

#### 解决方案
在执行前进行参数验证：

```rust
// 为每个工具添加 validate_parameters 方法
#[async_trait::async_trait]
pub trait Tool: Send + Sync {
    // ... existing methods ...
    
    /// Validate parameters before execution (optional override)
    fn validate_parameters(&self, args: &Value) -> Option<String> {
        None // Default: no validation
    }
}

// 在 FileReadTool 中实现
impl Tool for FileReadTool {
    // ... existing methods ...
    
    fn validate_parameters(&self, args: &Value) -> Option<String> {
        if let Some(path) = args.get("path").and_then(|p| p.as_str()) {
            if path.is_empty() {
                return Some("Parameter 'path' cannot be empty".to_string());
            }
            // Check for obviously invalid paths
            if path.contains('\0') {
                return Some("Parameter 'path' contains invalid characters".to_string());
            }
        }
        None
    }
}

// 在 agent/mod.rs 中调用验证
if let Some(validation_error) = tool.validate_parameters(&args) {
    let error_msg = format!(
        "⚠️ Parameter validation failed: {}\n\n\
         Please check your parameters and retry.",
        validation_error
    );
    // ... send error to UI and LLM ...
    continue;
}
```

### 方案 4: 智能重试机制 ⭐⭐

#### 问题
临时性错误（如网络超时、文件锁定）应该允许重试。

#### 解决方案
为可重试的错误添加自动重试：

```rust
const MAX_RETRIES: u32 = 2;

for tc in &tool_calls {
    let mut retry_count = 0u32;
    let mut last_error: Option<String> = None;
    
    while retry_count <= MAX_RETRIES {
        if retry_count > 0 {
            tracing::info!(
                "Retrying tool '{}' (attempt {}/{})",
                tc.name, retry_count, MAX_RETRIES
            );
            
            // Add retry hint to error message
            let _ = ui_tx.send(AgentToUiEvent::Status(format!(
                "Retrying {}... ({}/{})",
                tc.name, retry_count, MAX_RETRIES
            )));
        }
        
        let result = tool.execute(args.clone(), &tool_ctx).await;
        
        // Check if error is retryable
        if result.is_error && is_retryable_error(&result.content) && retry_count < MAX_RETRIES {
            tracing::warn!(
                "Retryable error for '{}': {}",
                tc.name, result.content
            );
            last_error = Some(result.content);
            retry_count += 1;
            
            // Wait before retry (exponential backoff)
            tokio::time::sleep(std::time::Duration::from_millis(500 * retry_count as u64)).await;
            continue;
        }
        
        // Success or non-retryable error
        break;
    }
    
    // If all retries failed, use last error
    if retry_count > MAX_RETRIES {
        if let Some(error) = last_error {
            // Handle final failure
        }
    }
}

fn is_retryable_error(error_msg: &str) -> bool {
    error_msg.contains("timed out") ||
    error_msg.contains("temporary") ||
    error_msg.contains("connection reset") ||
    error_msg.contains("resource busy")
}
```

### 方案 5: 超时时间动态调整 ⭐

#### 问题
固定 30s 超时不适合所有场景。

#### 解决方案
根据命令类型动态设置超时：

```rust
// 在 shell_exec.rs 中
let timeout_ms = args
    .get("timeout_ms")
    .and_then(|t| t.as_u64())
    .unwrap_or_else(|| {
        // Auto-detect appropriate timeout based on command
        if command.starts_with("git clone") || 
           command.starts_with("cargo build") ||
           command.starts_with("npm install") {
            120000 // 2 minutes for heavy operations
        } else if command.starts_with("ls") || 
                  command.starts_with("cat") ||
                  command.starts_with("echo") {
            5000 // 5 seconds for simple commands
        } else {
            30000 // 30 seconds default
        }
    });
```

### 方案 6: 工具调用统计和诊断 ⭐

#### 问题
无法了解哪些工具经常失败。

#### 解决方案
添加统计和诊断：

```rust
// 在 ToolRegistry 中添加统计
pub struct ToolRegistry {
    tools: HashMap<String, Box<dyn Tool>>,
    stats: Arc<Mutex<HashMap<String, ToolStats>>>,
}

#[derive(Debug, Clone)]
pub struct ToolStats {
    pub total_calls: u32,
    pub successful_calls: u32,
    pub failed_calls: u32,
    pub avg_execution_time_ms: f64,
    pub last_error: Option<String>,
}

impl ToolRegistry {
    pub fn get_stats(&self) -> HashMap<String, ToolStats> {
        let stats = self.stats.lock().unwrap();
        stats.clone()
    }
    
    pub fn record_success(&self, tool_name: &str, duration_ms: u64) {
        let mut stats = self.stats.lock().unwrap();
        let entry = stats.entry(tool_name.to_string()).or_insert(ToolStats {
            total_calls: 0,
            successful_calls: 0,
            failed_calls: 0,
            avg_execution_time_ms: 0.0,
            last_error: None,
        });
        
        entry.total_calls += 1;
        entry.successful_calls += 1;
        // Update running average
        entry.avg_execution_time_ms = 
            (entry.avg_execution_time_ms * (entry.total_calls - 1) as f64 + duration_ms as f64) 
            / entry.total_calls as f64;
    }
    
    pub fn record_failure(&self, tool_name: &str, error: &str) {
        let mut stats = self.stats.lock().unwrap();
        let entry = stats.entry(tool_name.to_string()).or_insert(ToolStats {
            total_calls: 0,
            successful_calls: 0,
            failed_calls: 0,
            avg_execution_time_ms: 0.0,
            last_error: None,
        });
        
        entry.total_calls += 1;
        entry.failed_calls += 1;
        entry.last_error = Some(error.to_string());
    }
}
```
