# 工具调用错误优化 - 实施完成

## 🎯 问题背景

大模型在调用工具时经常出错，主要问题包括：
1. **JSON 参数格式错误** - 生成的 JSON 不完整或格式不正确
2. **参数缺失** - 缺少必需的参数
3. **参数值错误** - 参数值为空或格式不对
4. **工具名称错误** - 使用了不存在的工具名称

这些问题导致工具调用失败，影响用户体验和任务完成率。

## 🔧 技术实现细节

### 1. 统一的错误消息格式

所有错误消息遵循统一模板：
```
❌ [错误类型]: [简要描述]

💡 How to fix:
• [建议 1]
• [建议 2]
• [建议 3]

📝 Example:
[具体示例]

Please retry with corrected [parameters/input].
```

### 2. 上下文感知的错误提示

根据不同工具提供针对性的示例：
```rust
let example = match tc.name.as_str() {
    "file_read" => "{\"path\": \"src/main.rs\", \"limit\": 100}",
    "file_write" => "{\"path\": \"output.txt\", \"content\": \"Hello World\"}",
    "shell_exec" => "{\"command\": \"ls -la\", \"timeout_ms\": 5000}",
    // ...
};
```

### 3. 智能建议生成

对于未知工具，基于前缀匹配推荐相似工具：
```rust
let mut suggestions: Vec<&str> = Vec::new();
for name in tool_registry.names() {
    if name.starts_with(&tc.name[..tc.name.len().min(3)]) ||
       tc.name.starts_with(&name[..name.len().min(3)]) {
        suggestions.push(name);
    }
}
```

## 💡 最佳实践建议

### 对 LLM 的建议
1. **生成 JSON 时**:
   - 使用双引号包裹键和字符串值
   - 不要在最后一项添加逗号
   - 检查括号配对

2. **调用工具前**:
   - 确认工具名称拼写正确
   - 检查所有必需参数都已提供
   - 验证参数格式符合要求

3. **遇到错误时**:
   - 仔细阅读错误提示中的修复建议
   - 参考提供的示例格式
   - 根据提示逐步修正

### 对开发者的建议
1. **添加新工具时**:
   - 提供清晰的 description
   - 定义详细的 parameters_schema
   - 添加完整的错误处理和示例

2. **调试工具调用**:
   - 查看 ~/.ox/logs/ox.log 日志文件
   - 关注 tracing::warn 和 tracing::error 输出
   - 使用详细的错误信息定位问题
