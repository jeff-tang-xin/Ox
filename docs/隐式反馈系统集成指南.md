# 隐式反馈系统集成指南

## 概述

本模块实现了 Ox CLI 的隐式反馈系统,包括:
1. **CodeOverrideDetector** - 监测用户是否修改了 Ox 生成的代码
2. **EMA Trend Tracker** - 使用指数移动平均追踪 accept_rate 等指标
3. **Rollback Manager** - 基于满意度评估的自动回滚机制

## 架构设计

```
┌─────────────────────────────────────────────────────┐
│              Main Loop (main.rs)                     │
│                                                      │
│  1. Before user input:                               │
│     ├─ Detect file overrides                         │
│     ├─ Calculate accept_rate                         │
│     └─ Update EMA tracker                            │
│                                                      │
│  2. After agent turn:                                │
│     ├─ Register new file writes                      │
│     ├─ Evaluate satisfaction                         │
│     └─ Create/rollback persona snapshots             │
└─────────────────────────────────────────────────────┘
         │                    │
         ▼                    ▼
┌─────────────────┐  ┌──────────────────┐
│ OverrideDetector│  │  EMA Manager      │
│                 │  │                   │
│ - recent_writes │  │ - trackers        │
│ - detect()      │  │ - update_metric() │
│ - register()    │  │ - persist/load()  │
└─────────────────┘  └──────────────────┘
                              │
                              ▼
                     ┌──────────────────┐
                     │ Rollback Manager │
                     │                  │
                     │ - create_snapshot│
                     │ - restore()      │
                     │ - evaluate()     │
                     └──────────────────┘
```

## 集成步骤

### Step 1: 在 App 结构中添加字段

```rust
// crates/ox-cli/src/main.rs

use ox_core::feedback::{
    CodeOverrideDetector, 
    Emamanager, 
    RollbackManager
};

struct App {
    // ... existing fields ...
    
    // Implicit feedback system
    override_detector: CodeOverrideDetector,
    ema_manager: Emamanager,
    rollback_manager: RollbackManager,
    
    // Tracking counters
    total_file_writes: u32,
    accepted_file_writes: u32,
    explicit_feedback_count: u32,
    good_feedback_count: u32,
}

impl App {
    fn new() -> Self {
        Self {
            // ... existing initialization ...
            
            override_detector: CodeOverrideDetector::new(300), // 5 min window
            ema_manager: Emamanager::new(0.2), // alpha = 0.2
            rollback_manager: RollbackManager::new(10), // max 10 snapshots per lang
            
            total_file_writes: 0,
            accepted_file_writes: 0,
            explicit_feedback_count: 0,
            good_feedback_count: 0,
        }
    }
}
```

### Step 2: 在文件写入时注册

```rust
// In handle_agent_event(), when receiving ToolResult for file_write:

AgentToUiEvent::ToolResult { name, output, is_error } => {
    if name == "file_write" && !is_error {
        // Extract file path and content from output
        if let Some((path, content)) = parse_file_write_output(&output) {
            app.override_detector.register_write(path, content);
            app.total_file_writes += 1;
        }
    }
    // ... rest of handling ...
}
```

### Step 3: 在用户输入前检测改动

```rust
// In the main loop, before reading user input:

loop {
    // Detect overrides before processing user input
    let override_signals = app.override_detector.detect_overrides();
    
    for signal in &override_signals {
        if let Some(feedback) = map_override_to_feedback(signal.change_ratio) {
            // Update counters based on feedback
            match feedback {
                ImplicitFeedback::WeakNegative => {
                    // Minor change - don't count as rejection
                }
                ImplicitFeedback::StrongNegative | 
                ImplicitFeedback::VeryStrongNegative => {
                    // Major rewrite or deletion - count as rejection
                    // (total already incremented, so just don't increment accepted)
                }
            }
            
            tracing::info!(
                "[IMPLICIT FEEDBACK] File {:?} changed by {:.1}% - {:?}",
                signal.path,
                signal.change_ratio * 100.0,
                feedback
            );
        } else {
            // No significant change - count as acceptance
            app.accepted_file_writes += 1;
        }
    }
    
    // Update EMA tracker with current accept_rate
    let accept_rate = app.ema_manager.calculate_accept_rate(
        app.total_file_writes,
        app.accepted_file_writes,
    );
    
    // Persist EMA state periodically (e.g., every 10 updates)
    if app.total_file_writes % 10 == 0 {
        let _ = app.ema_manager.persist_to_store(
            "code_accept_rate",
            &memory.overall_store(),
        );
    }
    
    // ... continue with user input handling ...
}
```

### Step 4: 计算满意度并评估回滚

```rust
// After TurnDone event, before persona evolution:

if config.persona.auto_evolve && !persona_vector.frozen {
    // Calculate composite satisfaction score
    let explicit_rate = if app.explicit_feedback_count > 0 {
        app.good_feedback_count as f64 / app.explicit_feedback_count as f64
    } else {
        0.5 // Neutral if no explicit feedback
    };
    
    let tool_success_rate = calculate_tool_success_rate(); // From your stats
    let code_accept_rate = app.ema_manager.get_value("code_accept_rate")
        .unwrap_or(0.8); // Default if not tracked yet
    
    let has_explicit = app.explicit_feedback_count >= 5;
    
    let satisfaction_score = app.rollback_manager.calculate_satisfaction_score(
        explicit_rate,
        tool_success_rate,
        code_accept_rate,
        has_explicit,
    );
    
    // Create snapshot before evolution
    let baseline_score = 0.75; // Could be loaded from persistent storage
    let decision = app.rollback_manager.evaluate_and_maybe_rollback(
        &mut persona_vector,
        satisfaction_score,
        baseline_score,
        &memory.overall_store(),
    ).unwrap_or_else(|e| {
        tracing::warn!("Rollback evaluation failed: {}", e);
        RollbackDecision::NoRollback { 
            current_score: satisfaction_score 
        }
    });
    
    match decision {
        RollbackDecision::RolledBack { from_score, to_score, snapshot_id } => {
            app.output.push_system(&format!(
                "Persona rolled back due to satisfaction drop ({:.2} → {:.2})",
                from_score, to_score
            ));
            tracing::info!("[ROLLBACK] Snapshot: {}, Score: {:.2} → {:.2}", 
                snapshot_id, from_score, to_score);
        }
        RollbackDecision::NeedsRollback { .. } => {
            app.output.push_system("Warning: Satisfaction dropped but no rollback point available");
        }
        RollbackDecision::NoRollback { current_score } => {
            // Normal evolution can proceed
            let evolution_log = memory.run_self_evaluation(
                &mut persona_vector,
                config.persona.max_trait_change,
            );
            // ... handle evolution log ...
        }
    }
}
```

### Step 5: 处理显式反馈

```rust
// In SlashCommand::Feedback handling:

SlashCommand::Feedback { kind } => {
    match kind.as_str() {
        "good" => {
            app.good_feedback_count += 1;
            app.explicit_feedback_count += 1;
            
            // Reinforce memories (existing logic)
            // ...
            
            app.output.push_system("Feedback noted: positive");
        }
        "bad" => {
            app.explicit_feedback_count += 1;
            // Don't increment good_feedback_count
            
            // Process feedback (existing logic)
            persona_vector.process_feedback("bad", config.persona.max_trait_change);
            
            app.output.push_system("Feedback noted: negative. Persona adjusted.");
        }
        _ => {
            app.output.push_system("Usage: /feedback <good|bad>");
        }
    }
}
```

## 配置建议

在 `config.toml` 中添加:

```toml

detection_window_secs = 300  # 5 minutes
ema_alpha = 0.2              # EMA smoothing factor
max_snapshots_per_lang = 10  # Max persona snapshots per language
rollback_threshold = 0.2     # Satisfaction drop threshold for rollback
```


### 常见问题

1. **Accept rate 始终为 1.0**
   - 检查是否正确调用了 `detect_overrides()` 在每次用户输入前
   - 确认 `register_write()` 在每次 `file_write` 后被调用

2. **回滚从未触发**
   - 检查 `baseline_satisfaction` 是否设置合理
   - 确认快照在演化前被正确创建

3. **EMA 趋势不稳定**
   - 调整 `alpha` 值: 较小 (0.1) = 更平滑但响应慢, 较大 (0.3) = 响应快但波动大

## 性能考虑

- **哈希计算**: 使用 `DefaultHasher`,对于大文件 (< 1MB) 通常在微秒级
- **检测频率**: 仅在用户输入前检测,不在后台轮询
- **存储开销**: 每个快照约 1KB,10 个快照 = 10KB,可忽略不计
