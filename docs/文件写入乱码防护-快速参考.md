# 文件写入乱码防护 - 快速参考

## 🎯 核心功能

在 `file_write` 和 `file_patch` 工具写入文件前，自动检测并阻止乱码内容。

##  5 层防护

| 检查 | 检测内容 | 阈值 | 错误提示 |
|------|---------|------|---------|
| **1. UTF-8 编码** | 无效编码序列 | - | Invalid UTF-8 encoding |
| **2. 不可打印字符** | 非可见字符比例 | >2% | Suspicious Content (X%) |
| **3. 乱码模式** | 连续特殊字符 | ≥5 或 ≥10 | Garbled Text Detected |
| **4. Null 字节** | `\x00` 空字节 | 任意 | Corrupted Content |
| **5. 替换字符** | U+FFFD `` | 任意 | Encoding Errors |

##  使用示例

### 正常代码（✅ 通过）

```rust
content = r#"fn main() {
    println!("Hello, 世界!");
    let x = 42;
}"#

// 结果: Written 100 bytes to src/main.rs ✅
```

### 乱码内容（❌ 阻止）

```rust
// LLM 错误生成的乱码
content = "fn main() {\u0000\u0000\u0000\u0000\u0000}"

// 结果: ❌ Garbled Text Detected
//        💡 Content appears to contain garbled characters
```

### 编码错误（❌ 阻止）

```rust
// GBK 误读为 UTF-8
content = "浣ソ\u{FFFD}\u{FFFD}"

// 结果: ❌ Encoding Errors Detected
//        💡 Content contains replacement characters (U+FFFD)
```

## 🚨 错误处理

LLM 收到错误后会：
1. 理解错误原因（详细的错误提示）
2. 重新生成正确的内容
3. 再次尝试写入

**不需要人工干预**，LLM 可以自动修正。
