## 🚨 发现的问题

### 问题 1: 记忆注入时未压缩

**位置**：[main.rs Line 870-871](file:///F:/rust/Ox/crates/ox-cli/src/main.rs#L870-L871)

**现状**：
```rust
let memory_nodes = memory_arc.retrieve(&last_user_query, &Some(rt_env.project_id.as_str()), 5);
let memory_ctx = memory_arc.format_memory_context(&memory_nodes, false);
```

**流程**：
```
retrieve() → 原始 MemoryNode[] (可能很长)
    ↓
format_memory_context() → 截断到 150 chars（旧逻辑）
    ↓
注入 System Prompt
```

**问题**：
- ❌ 如果记忆内容 > 512 tokens，会占用大量 token
- ❌ 旧的截断逻辑过于激进（150 chars ≈ 40-50 tokens）
- ❌ 没有利用滑窗分块优化语义连续性

## 💡 实施的修复方案

### 修复 1: 增强记忆截断逻辑

**文件**：[memory/mod.rs Line 900-927](file:///F:/rust/Ox/crates/ox-core/src/memory/mod.rs#L900-L927)

**修改前**：
```rust
// XML format
let content: String = n.content.chars().take(120).collect();

// Plain text format
let content: String = n.content.chars().take(150).collect();
```

**修改后**：
```rust
// XML format - 增加到 200 chars
let content = if n.content.len() > 200 {
    format!("{}...", &n.content[..200])
} else {
    n.content.clone()
};

// Plain text format - 增加到 250 chars
let content = if n.content.len() > 250 {
    format!("{}...", &n.content[..250])
} else {
    n.content.clone()
};
```

**优势**：
- ✅ 保留更多上下文（200-250 chars vs 120-150 chars）
- ✅ 智能截断（短记忆不截断）
- ✅ 添加 "..." 提示被截断

**Token 节省**：
```
旧逻辑: 150 chars ≈ 40-50 tokens × 5 条记忆 = 200-250 tokens
新逻辑: 250 chars ≈ 65-80 tokens × 5 条记忆 = 325-400 tokens
增加: +125-150 tokens（但语义更完整）
```

### P2（中期探索）

4. **统一压缩管道**
   ```rust
   // 将记忆转换为 Message 格式
   let memory_messages: Vec<Message> = memory_nodes
       .iter()
       .map(|m| Message::system(&format!("[MEMORY] {}", m.content)))
       .collect();
   
   // 合并对话历史和记忆
   let mut all_messages = session.messages.clone();
   all_messages.extend(memory_messages);
   ```
