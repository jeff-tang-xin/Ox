# 隐式反馈系统 - 快速参考

## API 速查

### CodeOverrideDetector

```rust
use ox_core::feedback::{CodeOverrideDetector, map_override_to_feedback};

// 创建检测器 (5分钟窗口)
let mut detector = CodeOverrideDetector::new(300);

// 注册 Ox 写入的文件
detector.register_write(path_buf, content_str);

// 检测用户改动 (在用户输入前调用)
let signals = detector.detect_overrides();

for signal in signals {
    if let Some(feedback) = map_override_to_feedback(signal.change_ratio) {
        println!("Change: {:.1}%, Feedback: {:?}", 
                 signal.change_ratio * 100.0, feedback);
    }
}
```

### EMA Manager

```rust
use ox_core::feedback::Emamanager;

// 创建管理器 (alpha=0.2)
let mut ema = Emamanager::new(0.2);

// 更新指标
ema.update_metric("code_accept_rate", 0.85);

// 便捷方法: 计算 accept_rate
let rate = ema.calculate_accept_rate(total_writes, accepted_writes);

// 持久化
ema.persist_to_store("code_accept_rate", &memory_store)?;

// 加载历史状态
ema.load_from_store("code_accept_rate", &memory_store)?;

// 查询当前值和趋势
let value = ema.get_value("code_accept_rate");
let trend = ema.get_trend("code_accept_rate");
```

### Rollback Manager

```rust
use ox_core::feedback::RollbackManager;

// 创建管理器 (每语言最多 10 个快照)
let rollback_mgr = RollbackManager::new(10);

// 创建快照
let snapshot_id = rollback_mgr.create_snapshot(
    &persona_vector,
    satisfaction_score,
    &memory_store,
)?;

// 评估并可能回滚
let decision = rollback_mgr.evaluate_and_maybe_rollback(
    &mut persona_vector,
    current_satisfaction,
    baseline_satisfaction,
    &memory_store,
)?;

match decision {
    RollbackDecision::RolledBack { from_score, to_score, .. } => {
        println!("Rolled back: {:.2} → {:.2}", from_score, to_score);
    }
    RollbackDecision::NoRollback { .. } => {
        println!("No rollback needed");
    }
    _ => {}
}

// 计算复合满意度
let satisfaction = rollback_mgr.calculate_satisfaction_score(
    explicit_feedback_rate,  // good / total
    tool_success_rate,       // success / total
    code_accept_rate,        // from EMA
    has_explicit_feedback,   // >= 5 feedbacks?
);
```

## 集成模板

```rust
// 1. App 结构
struct App {
    override_detector: CodeOverrideDetector,
    ema_manager: Emamanager,
    rollback_manager: RollbackManager,
    total_writes: u32,
    accepted_writes: u32,
}

// 2. 初始化
impl App {
    fn new() -> Self {
        Self {
            override_detector: CodeOverrideDetector::new(300),
            ema_manager: Emamanager::new(0.2),
            rollback_manager: RollbackManager::new(10),
            total_writes: 0,
            accepted_writes: 0,
        }
    }
}

// 3. 文件写入时
fn on_file_write(&mut self, path: PathBuf, content: &str) {
    self.override_detector.register_write(path, content);
    self.total_writes += 1;
}

// 4. 用户输入前
fn before_user_input(&mut self) {
    let signals = self.override_detector.detect_overrides();
    
    for signal in &signals {
        if let Some(feedback) = map_override_to_feedback(signal.change_ratio) {
            match feedback {
                ImplicitFeedback::WeakNegative => {} // Ignore minor changes
                _ => {} // Major changes: don't increment accepted
            }
        } else {
            // No significant change = accepted
            self.accepted_writes += 1;
        }
    }
    
    // Update EMA
    self.ema_manager.calculate_accept_rate(
        self.total_writes,
        self.accepted_writes,
    );
}

// 5. 对话结束后
fn after_turn(&mut self, persona: &mut PersonaVector, memory_store: &MemoryStore) {
    let satisfaction = self.rollback_mgr.calculate_satisfaction_score(
        self.explicit_rate(),
        self.tool_success_rate(),
        self.ema_manager.get_value("code_accept_rate").unwrap_or(0.8),
        self.has_enough_explicit_feedback(),
    );
    
    let decision = self.rollback_mgr.evaluate_and_maybe_rollback(
        persona,
        satisfaction,
        0.75, // baseline
        memory_store,
    ).ok();
    
    if let Some(RollbackDecision::RolledBack { .. }) = decision {
        println!("Persona rolled back!");
    } else {
        // Normal evolution
        memory.run_self_evaluation(persona, max_change);
    }
}
```

## 关键阈值

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `detection_window_secs` | 300 (5min) | 检测窗口时长 |
| `ema_alpha` | 0.2 | EMA 平滑因子 (0.1-0.3) |
| `max_snapshots_per_lang` | 10 | 每语言最大快照数 |
| `rollback_threshold` | 0.2 | 满意度下降阈值 |
| `change_ratio < 0.05` | 忽略 | 微调不视为负反馈 |
| `0.05 <= ratio < 0.30` | 弱负 | weight: 0.3 |
| `ratio >= 0.30` | 强负 | weight: 0.8 |
| `ratio == 1.0` (删除) | 极强负 | weight: 1.0 |

## 常见问题

**Q: Accept rate 始终为 1.0?**  
A: 确认在每次用户输入前调用了 `detect_overrides()`

**Q: 回滚从未触发?**  
A: 检查 `baseline_satisfaction` 是否合理设置,确认快照已创建

**Q: EMA 波动太大?**  
A: 降低 `alpha` 值 (如 0.2 → 0.1) 以平滑趋势

**Q: 内存占用过高?**  
A: 检查 `recent_writes` HashMap,过期记录会自动清理
