# 多模式智能体系统 - 快速参考

## 🚀 快速开始

### 1. 激活 Spec 模式

```rust
// 在 main.rs 中
app.activate_spec_mode("实现用户认证功能".to_string());
workflow_engine.activate_workflow("spec_workflow")?;
```

### 2. 检查工作流状态

```rust
// 检查是否允许工具调用
if !workflow_engine.allows_tool_execution() {
    return Err("Tools not allowed in current step");
}

// 检查是否需要用户确认
if workflow_engine.requires_user_confirmation() {
    // 显示确认对话框
}
```

### 3. 推进工作流

```rust
// 用户确认后推进
workflow_engine.advance_step()?;
```

## 📋 API 速查

### WorkflowEngine

```rust
// 创建引擎
let engine = WorkflowEngine::new(session_state);

// 激活工作流
engine.activate_workflow("spec_workflow")?;

// 获取当前步骤
let step = engine.current_step();

// 检查权限
engine.allows_tool_execution() -> bool
engine.requires_user_confirmation() -> bool

// 验证步骤
engine.validate_current_step() -> bool

// 推进步骤
engine.advance_step() -> Result<bool, String>

// 获取系统提示词
engine.get_step_system_prompt() -> Option<String>
```

### SessionState

```rust
// 创建 Session
let state = SessionState::new("session_1");

// 设置变量
state.set_variable("spec_confirmed", "true");

// 获取变量
state.get_variable("spec_confirmed") -> Option<&String>

// 检查变量
state.has_variable("spec_confirmed") -> bool

// 推进步骤
state.advance_step(total_steps) -> bool

// 等待确认
state.wait_for_confirmation()
state.clear_confirmation()
```

### InterventionManager

```rust
// 创建管理器
let manager = InterventionManager::new();

// 队列介入请求
manager.queue_intervention(request);

// 激活下一个请求
manager.activate_next() -> Option<&InterventionRequest>

// 处理用户响应
let action = manager.process_response(response);

// 清除所有请求
manager.clear_all()
```

## 🔧 创建工作流

### 定义步骤

```rust
let mut workflow = Workflow::new("my_workflow", "My Custom Workflow");

// 普通步骤
workflow.add_step(
    WorkflowStep::new("step1", "Step 1", "Description")
);

// 需要用户确认的步骤
workflow.add_step(
    WorkflowStep::new("step2", "Step 2", "Description")
        .require_confirmation()
);

// 禁止工具调用的步骤
workflow.add_step(
    WorkflowStep::new("step3", "Step 3", "Description")
        .disallow_tools()
);

// 带验证器的步骤
workflow.add_step(
    WorkflowStep::new("step4", "Step 4", "Description")
        .with_validator("check_file_exists")
);

// 带系统提示词的步骤
workflow.add_step(
    WorkflowStep::new("step5", "Step 5", "Description")
        .with_prompt("Custom prompt for this step")
);
```

### 注册验证器

```rust
state_registry.register_validator("check_file_exists", |state| {
    state.has_variable("file_created")
});
```

## 🎯 三种默认工作流

### Spec Mode (6步)

```rust
let workflow = create_spec_workflow();
// Steps:
// 1. RequirementAnalysis
// 2. GeneratingSpec
// 3. AwaitingSpecConfirmation (requires confirmation, no tools)
// 4. GeneratingTask
// 5. AwaitingTaskConfirmation (requires confirmation, no tools)
// 6. Executing
```

### Council Mode (6步)

```rust
let workflow = create_council_workflow();
// Steps:
// 1. TopicDefinition (requires confirmation, no tools)
// 2. ProposalPhase (no tools)
// 3. ReviewPhase (no tools)
// 4. RebuttalPhase (no tools)
// 5. Arbitration (no tools)
// 6. Conclusion (requires confirmation)
```

### Free Mode (1步)

```rust
let workflow = create_free_workflow();
// Steps:
// 1. FreeInteraction (full access)
```

## 💡 常见场景

### 场景 1: 阻止工具调用

```rust
// 在 agent turn 开始前检查
if !workflow_engine.allows_tool_execution() {
    ui_tx.send(AgentToUiEvent::Status(
        "Tool execution not allowed in current step".to_string()
    ))?;
    return Ok(());
}
```

### 场景 2: 请求用户确认

```rust
if workflow_engine.requires_user_confirmation() {
    let request = workflow_engine.request_confirmation(
        "Please confirm to proceed"
    );
    
    if let Some(req) = request {
        intervention_manager.queue_intervention(req);
        return Ok(()); // 等待用户响应
    }
}
```

### 场景 3: 处理用户响应

```rust
match user_response {
    InterventionResponse::Confirmed => {
        let action = intervention_manager.process_response(
            InterventionResponse::Confirmed
        );
        
        match action {
            InterventionAction::Proceed => {
                workflow_engine.advance_step()?;
                // 继续执行
            }
            _ => { /* 处理其他动作 */ }
        }
    }
    _ => { /* 处理其他响应 */ }
}
```

### 场景 4: 动态系统提示词

```rust
// 构建系统提示词时加入当前步骤的提示
let mut system_prompt = build_base_system_prompt();

if let Some(step_prompt) = workflow_engine.get_step_system_prompt() {
    system_prompt.push_str("\n\n## Current Step Instructions\n\n");
    system_prompt.push_str(&step_prompt);
}
```

### 场景 5: 切换 Session

```rust
// 保存当前 Session
state_registry.create_session("session_1");

// 切换到新 Session
if let Some(new_session) = state_registry.switch_session("session_2") {
    // 更新 UI
    app.workflow_state = new_session.current_mode.parse()?;
}
```

## ⚠️ 注意事项

### 1. Deprecated 字段

以下字段已标记为 deprecated，请迁移到新 API：

```rust
// ❌ 旧方式（deprecated）
app.spec_content = "...";
app.spec_active = true;

// ✅ 新方式
app.activate_spec_mode("...".to_string());
```

### 2. 线程安全

`SessionState` 和 `InterventionManager` 使用 `Arc<Mutex<>>` 包装：

```rust
let session_state = Arc::new(tokio::sync::Mutex::new(SessionState::new("id")));
let intervention_manager = Arc::new(tokio::sync::Mutex::new(InterventionManager::new()));
```

访问时需要 `.lock().await` 或 `.blocking_lock()`。

### 3. 验证器命名

验证器名称必须与注册的名称完全匹配：

```rust
// 注册
state_registry.register_validator("check_spec_exists", ...);

// 使用
WorkflowStep::new("...", "...").with_validator("check_spec_exists")
//                                  ^^^^^^^^^^^^^^^^^^ 必须完全匹配
```

## 🐛 调试技巧

### 1. 查看当前工作流状态

```rust
tracing::info!(
    "Current workflow: {:?}, Step: {}/{}",
    workflow_engine.current_workflow().map(|w| &w.name),
    session_state.lock().await.current_step_index + 1,
    workflow_engine.current_workflow().map(|w| w.total_steps()).unwrap_or(0)
);
```

### 2. 查看所有 Session

```rust
for (id, state) in state_registry.sessions.iter() {
    tracing::info!(
        "Session {}: mode={}, step={}, active={}",
        id,
        state.current_mode,
        state.current_step_index,
        state.is_active
    );
}
```

### 3. 检查验证器

```rust
let is_valid = workflow_engine.validate_current_step();
tracing::info!("Current step valid: {}", is_valid);
```
