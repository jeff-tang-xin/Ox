# 关键词提取与语义学习诊断日志

## 概述

本次更新添加了详细的日志输出，用于诊断关键词提取、语义学习和记忆检索的问题。

## 使用指南

### 场景 1: 检查关键词提取是否工作

查看日志中是否有：
```
[KEYWORD EXTRACTION] ✅ Extracted ... keywords
```

如果没有，说明 LLM 没有输出 JSON 代码块，需要修改 System Prompt。

### 场景 2: 检查语义学习是否记录

查看日志中是否有：
```
[SEMANTIC LEARNING] ✅ Recorded ... keywords
```

如果看到：
```
[SEMANTIC LEARNING] ⚠️ Semantic manager not initialized
```
说明语义管理器初始化失败，需要检查数据库连接。

### 场景 3: 检查查询扩展是否生效

查看日志中是否有：
```
[SEMANTIC EXPANSION] ✅ 'query' → N related terms
```

如果总是看到：
```
[SEMANTIC EXPANSION] ⚠️ No related terms found
```
说明语义关联数据库为空，需要先积累数据。

### 场景 4: 诊断记忆检索返回 0

查看完整的检索流程日志：

1. **PATH 1-4 是否有结果？**
   ```
   [MEMORY RETRIEVAL] PATH 1 (query): 'xxx' → 0 results
   [MEMORY RETRIEVAL] PATH 2 (entity 'yyy'): 0 results
   [MEMORY RETRIEVAL] PATH 3 (type-specific): 0 results
   [MEMORY RETRIEVAL] PATH 4 (global): 0 results
   ```

2. **总候选数是多少？**
   ```
   [MEMORY RETRIEVAL] Total candidates before decay filter: 0
   ```

3. **是否被衰减过滤掉了？**
   ```
   [MEMORY RETRIEVAL] Decay filter: 5 → 0 memories (removed 5)
   ```

4. **最终结果**
   ```
   [MEMORY RETRIEVAL] Initial retrieval returned 0 memories
   ```

### 场景 5: 检查重排序是否启用

查看日志：
```
[MEMORY RETRIEVAL] Starting retrieval for query: '...', limit: 5, rerank: enabled
```

如果显示 `rerank: disabled`，说明 BGE Embedder 未配置或未启用。

## 常见问题诊断

### 问题 1: 关键词从未提取

**症状**: 日志中只有 `⚠️ No JSON block found`

**原因**: LLM 没有在响应中输出 ```json 代码块

**解决**: 修改 System Prompt，强制要求 LLM 输出关键词

---

### 问题 2: 语义管理器未初始化

**症状**: 日志中出现 `Semantic manager not initialized`

**原因**: 
- 数据库文件不存在
- 表结构未创建
- 数据库连接失败

**解决**: 
1. 检查 `~/.ox/db/memories_overall.db` 是否存在
2. 检查数据库初始化代码
3. 查看错误日志

---

### 问题 3: 记忆检索始终返回 0

**可能原因**:
1. 数据库中确实没有记忆
2. 查询词与记忆内容不匹配
3. 所有记忆的衰减分数 < 0.3

**诊断步骤**:
1. 查看 PATH 1-4 的结果数
2. 查看衰减过滤前后的数量
3. 检查数据库中是否有数据：
   ```bash
   sqlite3 ~/.ox/db/memories_overall.db "SELECT COUNT(*) FROM memories;"
   ```

---

### 问题 4: 查询扩展无效

**症状**: 日志显示 `No related terms found`

**原因**: 语义关联数据库为空（因为关键词从未提取）

**解决**: 先确保关键词提取工作正常，积累数据后查询扩展才会生效

---

## 下一步行动

根据日志输出，确定问题所在：

1. **如果关键词提取失败** → 修改 System Prompt
2. **如果语义管理器未初始化** → 检查数据库
3. **如果记忆数据库为空** → 等待数据积累或手动导入
4. **如果衰减过滤太严格** → 调整阈值（当前是 0.3）
5. **如果重排序未启用** → 检查 BGE 配置

---

## 日志级别说明

- **INFO**: 关键事件（成功提取、成功记录、检索结果）
- **DEBUG**: 详细信息（具体关键词、实体列表、各路径结果）
- **WARN**: 警告信息（未找到 JSON、语义管理器未初始化）
- **ERROR**: 错误信息（数据库操作失败）

建议运行时设置日志级别为 `INFO` 以查看关键信息：
```bash
RUST_LOG=info cargo run
```

如需详细调试，设置为 `DEBUG`:
```bash
RUST_LOG=debug cargo run
```
