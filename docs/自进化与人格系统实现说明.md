# 自进化与人格系统实现说明

## 概述

本次实现完成了 Ox CLI 的自进化和人格系统的完整集成,包括**显式反馈机制**和**被动演化机制**。

## 已实现功能

### 1. 显式用户反馈 (`/feedback` 命令)

#### `/feedback good`
- **作用**: 强化最近使用的记忆节点
- **实现**: 
  - 检索最后一条用户消息相关的记忆
  - 增加这些记忆的 `depth` 和更新 `last_accessed`
  - 提升记忆在后续检索中的权重

#### `/feedback bad`
- **作用**: 触发人格演化调整
- **实现**:
  - 检测最近的助手回复内容
  - 调用 `persona_vector.process_feedback()` 分析反馈类型
  - 自动调整人格向量(如简洁度、风格严格度等)
  - 持久化保存到 SQLite

#### `/feedback unsafe`
- **作用**: 记录安全违规(但 `refuses_unsafe_code` 字段不可修改)
- **实现**: 记录到日志,供后续审计

### 2. 被动演化机制 (自动触发)

#### 每轮对话后自动演化
在每次 `TurnDone` 事件处理中:
```rust
if config.persona.auto_evolve && !persona_vector.frozen {
    let evolution_log = memory.run_self_evaluation(
        &mut persona_vector,
        config.persona.max_trait_change,
    );
    // ... 通知用户演化结果
}
```

#### 演化逻辑 (`analyze_and_evolve_persona`)
基于记忆模式自动调整人格:

1. **Style 记忆分析**:
   - 如果 >5 条 Style 记忆强调"简洁" → 提高 `prefers_conciseness`
   - 如果 >5 条 Style 记忆强调"规范" → 提高 `code_style_strictness`

2. **AntiPattern 记忆分析**:
   - 如果 >3 条 AntiPattern 涉及安全问题 → 提高 `favors_safety_over_speed`

3. **持久化**:
   - 所有演化结果自动保存到 `.ox/db/memories_overall.db` 的 `persona_vectors` 表

### 3. 人格管理命令 (`/persona`)

#### `/persona show`
显示当前人格向量状态:
```
Persona: Safety: 0.90 | Conciseness: 0.80 | Style: 0.90 | RefusesUnsafe: true | Lang: rust | Frozen: false
Auto-evolve: true | Max change: 0.1
```

#### `/persona freeze`
冻结人格,停止所有自动演化(包括显式和被动)

#### `/persona unfreeze`
解冻人格,恢复自动演化

#### `/persona evolve` (新增)
手动触发一次自我评估和演化,不依赖用户反馈

### 4. 配置控制

在 `config.toml` 中:
```toml
[persona]
auto_evolve = true          # 是否启用自动演化
max_trait_change = 0.1      # 单次演化最大变化幅度
frozen = false              # 是否冻结人格
export_format = "json"      # 导出格式(预留)
```

## 技术细节

### 演化约束
1. **安全字段锁定**: `refuses_unsafe_code` 永远不能被演化修改
2. **变化上限**: 单次演化不超过 `max_trait_change`(默认 0.1)
3. **范围限制**: 所有数值字段保持在 [0.0, 1.0] 区间
4. **冻结保护**: `frozen = true` 时禁止所有演化

### 数据存储
- **位置**: `.ox/db/memories_overall.db`
- **表名**: `persona_vectors`
- **字段**: language, safety_over_speed, prefers_conciseness, code_style_strictness, refuses_unsafe_code, frozen, forbidden_phrases, moral_priorities, last_updated

### 演化触发条件
| 触发方式 | 频率 | 说明 |
|---------|------|------|
| `/feedback bad` | 用户主动 | 立即调整人格 |
| 每轮对话后 | 自动 | 分析记忆模式,满足阈值才演化 |
| `/persona evolve` | 用户主动 | 强制触发一次评估 |

## 示例场景

### 场景 1: 用户觉得回复太啰嗦
```bash
# 用户输入
/feedback bad

# 系统响应
Feedback noted: negative. Persona adjusted.

# 后台发生
- persona_vector.prefers_conciseness += 0.05 (最大 0.1)
- 保存到 SQLite
- 下次对话时会使用更简洁的风格
```

### 场景 2: 多次代码安全问题
```bash
# 假设记忆中积累了多条安全相关的 AntiPattern
# 下一轮对话结束后自动触发

# 系统输出
Persona evolved: 1 adjustments
  → Evolved towards safer code (5 safety issues found)

# 后台发生
- persona_vector.favors_safety_over_speed += 0.05
- 保存并应用到后续对话
```

### 场景 3: 查看当前人格状态
```bash
/persona show

# 输出
Persona: Safety: 0.95 | Conciseness: 0.85 | Style: 0.90 | RefusesUnsafe: true | Lang: rust | Frozen: false
Auto-evolve: true | Max change: 0.1
```

## 设计原则

1. **最小干预**: 演化是渐进的,不会突然大幅改变行为
2. **可逆性**: 用户可以通过 `/persona freeze` 随时停止演化
3. **透明性**: 每次演化都会通知用户具体调整内容
4. **安全性优先**: 安全相关字段(`refuses_unsafe_code`)永远不可被演化修改
5. **证据驱动**: 被动演化需要足够的记忆模式证据(如 >5 条相似记忆)

## 未来扩展

可以进一步增强的方向:
1. **隐式反馈信号**: 检测用户对生成代码的修改率(code_accept_rate)
2. **EMA 趋势追踪**: 使用指数移动平均追踪长期演化趋势
3. **回滚机制**: 如果演化后满意度下降,自动回滚到之前的人格状态
4. **多语言人格隔离**: 为不同编程语言维护独立的人格向量
